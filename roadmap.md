### Что требуется от владельца проекта для реализации Монетизации и Бэкенда:
1. **Для бэкенда (Selectel):**
   - Доступ по SSH к серверу (IP, логин, пароль/ключ) или подготовленное окружение (например Docker).
   - API-ключ Gemini (или Google AI Studio / Vertex AI) для подстановки в сервер.
   - Файл ключа Firebase (`firebase_key.json`) для управления авторизацией.
2. **Для РСЯ (Реклама):**
   - ID приложения (Application ID, например `R-M-XXXXXX`).
   - ID рекламного блока с видео с вознаграждением (Rewarded Video Unit ID).
3. **Для RuStore (Подписки):**
   - ID подписки, созданный в консоли разработчика RuStore.
   - `RUSTORE_KEY_ID` и `RUSTORE_COMPANY_ID` для серверной валидации покупок.
   - `RUSTORE_PRIVATE_KEY` (приватный ключ) для подписи запросов сервер-сервер.
4. **Для Google Play (на будущее):**
   - JSON-ключ сервисного аккаунта Google Play Console (потребуется позже).
